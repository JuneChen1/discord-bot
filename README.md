# 🔔 Discord Reminder Bot 指令手冊

歡迎使用 Discord 提醒機器人！本專案提供單次與週期性提醒功能，並支援 CSV 批次匯入。

---

## 🛠️ 指令說明

### 1. `/remind` —— 建立單次提醒

設定一個特定事件的提醒。系統會將提醒自動發送到指定頻道。

- **發送日期**：預設為事件前一天（可自訂 `remind_date`）。
- **發送時間**：預設為你的個人設定（未設定則為台灣時間 `22:00`，可自訂 `remind_time`）。

| 參數          |  必填  | 格式 / 範例                  | 說明                               |
| :------------ | :----: | :--------------------------- | :--------------------------------- |
| `date`        | **是** | `YYYYMMDD` (e.g. `20260510`) | 事件發生的日期                     |
| `message`     | **是** | 文字                         | 提醒的內容通知                     |
| `time`        |   否   | `HH:MM` (e.g. `14:30`)       | 事件發生的具體時間                 |
| `remind_time` |   否   | `HH:MM`                      | 提醒發送的時間（預設套用個人設定） |
| `remind_date` |   否   | `YYYYMMDD`                   | 提醒發送的日期（預設為事件前一天） |

> 💡 **防重複機制**：同一日期時間若已有相同內容的提醒，系統將不會重複建立。

---

### 2. `/remind-recurring` —— 建立週期提醒

依據特定頻率重複發送提醒。建立時 **`end_date` 與 `count` 必須擇一填寫**。

| 參數                 |  必填  | 格式 / 說明                                                          |
| :------------------- | :----: | :------------------------------------------------------------------- |
| `type`               | **是** | 週期類型，可選：`daily`（每天）／`weekly`（每週）／`monthly`（每月） |
| `date`               | **是** | `YYYYMMDD`，**第一次**事件發生的日期                                 |
| `message`            | **是** | 提醒的內容通知                                                       |
| `time`               |   否   | `HH:MM`，事件發生的時間                                              |
| `remind_time`        |   否   | `HH:MM`，提醒發送時間（預設套用個人設定）                            |
| `remind_offset_days` |   否   | 提醒**提前幾天**發送，預設為 `1` 天，填 `0` 代表當天提醒（每次重複皆會套用）      |
| `end_date`           |  否\*  | `YYYYMMDD`，重複截止日期（與 `count` 二選一）                        |
| `count`              |  否\*  | 正整數，總共要提醒的場次數（與 `end_date` 二選一）                   |

- **對齊邏輯**：
  - `weekly` 依據 `date` 的「星期幾」重複。
  - `monthly` 依據 `date` 的「日期」重複（若遇到大月 31 號、小月無此日期時，系統會自動調整為當月最後一天發送）。
- **運行方式**：每次提醒成功發送後，系統會自動計算並排程下一個場次，直到觸發結束條件。

---

### 3. `/remind-import` —— CSV 批次匯入

從 CSV 檔案批次匯入多筆提醒，支援同時混用「單次提醒」與「週期提醒」。

| 參數   |  必填  | 說明                                                                                                 |
| :----- | :----: | :--------------------------------------------------------------------------------------------------- |
| `file` | **是** | 請上傳符合格式的 CSV 檔案。詳細範例請參考 [reminders_template.csv](templates/reminders_template.csv) |

- **欄位順序**：`date`, `message`, `time`, `remind_time`, `remind_date`, `type`, `remind_offset_days`, `end_date`, `count`（後 8 欄為選填）。
- **判斷邏輯**：
  - 若 `type` 欄位有填寫（`daily`/`weekly`/`monthly`），自動識別為**週期提醒**，此時會改用 `remind_offset_days`，且須提供 `end_date` 或 `count`。
  - 若 `type` 留空，則視為**單次提醒**，維持 `remind_date` 的傳統用法。
  - 同一日期時間若已有相同內容的提醒，該列將**跳過**並在匯入結果中標示失敗。

### 4. 管理與修改指令

#### 💾 `/remind-default` (個人預設時間設定)

查看或修改未指定 `remind_time` 時的系統預設發送時間。

- 參數 `time` (否)：設定新的預設時間，格式 `HH:MM`（例如 `21:00`）。
- 參數 `reset` (否)：重設回系統預設的 `22:00`（布林值）。
- _註：不帶參數時可查閱當前設定；`time` 與 `reset` 互斥不可同時使用。_

#### 🔍 `/reminders` (查看所有提醒)

分頁列出你個人所有「待發送」的提醒（包含事件日期、提醒時間、內容、頻道與 **ID**）。

- 一次最多顯示 20 筆（可透過 `lib/config.json` 的 `maxRemindersDisplay` 調整，上限為 Discord 硬限制 25 筆）。若超出上限，請改用 `/reminders-range` 縮小範圍。
- 週期性提醒會在列表前方標示 🔁 符號（例如：`🔁 每週一`）。

#### 📅 `/reminders-range` (區間查詢)

查詢特定事件日期區間內的提醒，一次最多顯示 20 筆（同 `/reminders` 上限）。

- 參數 `from` (**是**)：起始日期 `YYYYMMDD`。
- 參數 `to` (否)：結束日期 `YYYYMMDD`（留空則僅查詢 `from` 當天）。
- _註：若查詢區間已完全過去（`to` 早於台灣今日），系統會回覆「查詢區間已過期」並拒絕執行。_

#### ✏️ `/remind-edit` (編輯提醒)

透過 **ID** 變更現有的提醒設定，至少需提供一個要修改的欄位（`message`/`date`/`time`/`remind_date`/`remind_time`）。

- _註：只能編輯自己建立的提醒；目前暫不支援直接修改週期提醒的「週期規則變更」。_

#### ❌ `/remind-delete` (刪除提醒)

透過 **ID** 刪除一個或多個待發送的提醒。

- 參數 `id` (**是**)：提醒 ID（可從建立時的訊息底部或 `/reminders` 查詢）。若要刪除多個，請**用空白隔開**（例如：`id_001 id_002`）。
- _註：若刪除的是週期性提醒，將會直接終止該系列的後續所有排程。_

#### ❓ `/help`

在 Discord 頻道中即時列出所有可用指令與簡短說明。

## 💻 本地端部署與運作說明

### 🚀 啟動步驟

1. **配置環境變數**
   複製專案根目錄的 `.env.example` 並重新命名為 `.env`，填入對應資訊：
   ```env
   DISCORD_TOKEN=你的_Discord_Bot_Token
   REMINDER_CHANNEL_ID=（選填）固定發送提醒的頻道_ID
   DATA_DIR=（選填）資料儲存路徑
   ```
2. **安裝相依套件與啟動服務**


   ```
   # 安裝套件
   npm install

   # 啟動 Bot
   npm start
   ```
---

### ⚙️核心運作邏輯
* **重啟重排程**：機器人啟動時，會自動讀取資料庫並將所有「尚未到期」的提醒重新載入排程。
* **過期清理**：若提醒的發送時間早於機器人的啟動時間，該筆資料會在啟動時自動被判定**過期並刪除**。
* **資料儲存機制**：
  * **提醒事件庫**：預設儲存於根目錄的 `reminders.json`（可透過 `DATA_DIR` 更改）。
  * **使用者設定**：個人預設時間儲存於同一目錄下的 `user_settings.json`。
* **固定頻道發送**：預設狀況下，提醒會發送到當初執行建立指令的文字頻道。若在 `.env` 中設定了 `REMINDER_CHANNEL_ID`，則不論在哪裡建立指令，**所有提醒一律會導向該固定頻道**。
* **手動操作**：如果你需要大量且複雜的排程，亦可在關閉 Bot 的狀態下直接手動編輯 `reminders.json`，其欄位與匯入 CSV 一致，重啟 Bot 後排程便會自動生效。
